# SISTEMA

## 📌 Descripción del Proyecto
Este proyecto consiste en el desarrollo de una aplicación web realizada como parte del laboratorio de Introducción al Control de Versiones con Git y GitHub.  
El objetivo principal fue aplicar los conceptos fundamentales de control de versiones, manejo de repositorios, commits y ramas (branching).

La aplicación permite la navegación entre diferentes páginas web, simulando un sistema básico con funcionalidades de interacción para el usuario.

---

## 🎯 Objetivos del Proyecto
- Aplicar el uso de Git como sistema de control de versiones
- Gestionar cambios mediante commits descriptivos
- Implementar el uso de ramas para trabajo paralelo
- Publicar un proyecto en GitHub
- Documentar el proyecto mediante un archivo README

---

## 🛠 Tecnologías Utilizadas
- HTML5
- CSS3
- JavaScript
- PHP

---

## ⚙️ Funcionalidades
- Navegación entre múltiples páginas web
- Página principal con contenido informativo
- Formulario de contacto funcional
- Visualización de productos
- Secciones organizadas por categorías
- Simulación de proceso de compra

---

## 🌿 Control de Versiones
Durante el desarrollo del proyecto se utilizaron buenas prácticas de Git, incluyendo:

- Uso de commits descriptivos (feat, fix, docs)
- Creación de ramas:
  - desarrollo
  - pruebas
- Eliminación de ramas innecesarias
- Sincronización con repositorio remoto en GitHub

---

## 📂 Estructura del Proyecto
- `/css` → estilos de la aplicación
- `/js` → scripts JavaScript
- `/imagen` → recursos gráficos
- `/fuentes` → tipografías
- Archivos `.html` → páginas del sistema
- Archivos `.php` → procesamiento de formularios
